OBSCURA is a zero-knowledge file-transfer service. Files are encrypted in your browser with AES-256-GCM before they ever leave the page. The server stores ciphertext only — encryption keys never touch the wire — and the share link self-destructs after a configurable read count or TTL.

▸ send a file how it works ↓
01 · encrypt
in your browser
A random 256-bit key is generated in-page. Your files are packed and run through AES-256-GCM via WebCrypto. Optional Argon2id passphrase wrapping.
02 · upload
ciphertext only
The encrypted blob goes to a Cloudflare Worker. The key stays in the URL fragment (#k=…) — browsers never send fragments to servers.
03 · burn
TTL or read-count
Configurable expiry (up to 7 days) and download cap (1–100). When either fires, R2 is purged and KV forgets. You can also burn manually.

✓ what we can see

  • Opaque ciphertext bytes
  • Total ciphertext size
  • Aggregate daily counters (no per-share data)
  • Cloudflare-side IP logs (see privacy)

✗ what we never see

  • Plaintext bytes — encrypted before upload
  • Encryption keys — live in the URL fragment
  • Sender or recipient identity — no accounts, no email
  • Filenames or content metadata in the clear
