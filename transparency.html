methodology
Counters are stored as daily KV keys (stat:YYYY-MM-DD:<event>) with a 90-day TTL. Increments are best-effort read-then-write — two events landing in the same second can race and undercount by one. The counters carry no per-share data, no IPs, no identifiers; you can reconstruct them from git log + the application source. Raw JSON: /api/transparency. Full data policy: privacy.
